# 🏎️ F1 Bayesian Predictor & Live Tracker 2026

[![Streamlit App](https://img.shields.io/badge/Streamlit-FF4B4B?style=flat&logo=streamlit&logoColor=white)](https://f1predict.streamlit.app/)
[![Python 3.10+](https://img.shields.io/badge/python-3.10+-red.svg)](https://www.python.org/)
[![FastF1](https://img.shields.io/badge/API-FastF1-orange.svg)](https://github.com/theOehrly/FastF1)

An advanced predictive analytics platform for simulating Formula 1 qualifying and race events in real-time under the all-new 2026 engine & chassis regulations. It combines a **Hybrid ML Architecture** — LightGBM *Learning-To-Rank* trained on 2022-2025 structural patterns, **Season ELO Carryover** accumulating driver ratings across completed GPs, **Dynamic Constructor Pace** inference from real qualifying data, and a NumPy-vectorized Monte Carlo physics simulator with 10,000+ iterations.

---

## 🌟 Key Features

### 1. ⏱️ ML-Based Qualifying Grid Predictor
* **Algorithm**: `LightGBM LTR (Learning-To-Rank)` powered by *LambdaMART* to accurately rank starting grid positions.
* **Bayesian Credible Intervals**: Uses *Quantile Regression* to deliver best-case, median, and worst-case lap time estimates within a dynamic 90% credible interval sensitive to track temperature and rain intensity.
* **SHAP Interpretability**: Visual breakdown of how practice session performance (FP3/SQ) and driver prior factors contribute to the predicted outcome.

### 2. 🏁 Vectorized Monte Carlo Race Simulator (10,000+ Runs)
* **Lap-by-Lap Physics Simulation**: Simulates thousands of alternative races stochastically in under 1 second using fully vectorized NumPy matrix operations.
* **Realistic Race Modelling**: Integrates physical tyre degradation models (Soft, Medium, Hard), dirty air effects, circuit-history-based Safety Car probabilities, random DNF crashes, and pit stop strategy.
* **DNF Enforcement**: Guarantees that drivers confirmed retired at a specific lap carry a 100% DNF probability in simulation results.

### 3. 🔴 Live Status Awareness & Pipeline Behavior by Session State
The system automatically classifies each Grand Prix into one of three dynamic states based on real-time UTC timestamps from the official FastF1 2026 schedule, modifying the data pipeline as follows:

| Session State | FP3 Data Source | Qualifying & Starting Grid | Monte Carlo Race Simulator | UI Features & Refresh |
| :--- | :--- | :--- | :--- | :--- |
| **`📅 UPCOMING`** *(Before GP Weekend starts)* | **Calibrated Fallback Generator**: Synthesizes realistic times using: `Base Time = Circuit Length × 14.5s` + ELO-pace offsets + driver variance. | Uses **ML Predicted Starting Grid** generated by LightGBM Ranker (85% ML Ranker + 15% Weekend Practice Form). | Simulates the **entire race** (from Lap 0 to the final lap) starting from the predicted qualifying grid. | Static layout; no background API polls; countdown timer shown. |
| **`🔴 ONGOING / LIVE`** *(During live session)* | **FastF1 Live Timing**: Ingests live practice telemetry, tire compounds, and sector times as they occur. | Adapts predictions as live practice times populate. | Performs **mid-race simulations** dynamically starting from the **current lap**. Fetches real-time driver intervals, tire compound types/ages, and **enforces 100% DNF probability** for retired drivers. | 30-second background auto-refresh via `@st.fragment`; live dashboard view. |
| **`✅ COMPLETED / END`** *(Post-race completion)* | **FastF1 Historical API**: Pulls final practice session logs from cached databases. | Reconstructs the actual Saturday Qualifying classification using historical records. | Replays the race. Monte Carlo simulator is locked or can simulate alternative outcomes from any historical lap point. | Full access to **Lap Replay Slider** (Lap 0 → Finish) to inspect historical telemetry, tire history, and gaps. |

### 4. 🔄 Sprint Weekend Dynamic Fallbacks
* **Smart Session Routing**: If the selected circuit runs a *Sprint Weekend* format (which omits the standard FP3 session), the ingestion module automatically routes queries to **Sprint Qualifying (SQ)** or **Practice 1 (FP1)**, ensuring the ELO-updating and ranking pipeline executes without crashing.

### 5. 📈 Season ELO Carryover
* **Cumulative Driver Ratings**: Instead of static base ELO, driver ratings are dynamically accumulated across all completed 2026 GPs, using Qualifying results (60% weight) and Race results (40% weight) with K-factor=24.
* **Progressive Accuracy**: Predictions improve as the season progresses — each completed GP adds more data to the ELO accumulation.
* **Pre-computed Cache**: Season ELO values are pre-computed and cached to `data/season_elo_cache.json` for instant loading on Streamlit Cloud.

### 6. 🔧 Dynamic Constructor Pace Inference
* **Upgrade Detection**: Constructor pace offsets are dynamically inferred from real qualifying gap-to-pole data across completed 2026 GPs, using a rolling 3-GP window.
* **Trend Analysis**: Automatically classifies each team's performance trend as `improving`, `stable`, or `declining` based on recent results.
* **Replaces Static Offsets**: Overrides the hardcoded `CONSTRUCTORS_2026` pace offsets with data-driven values from real qualifying sessions.

## 🏛️ Bayesian Mathematical Architecture

The system applies **Bayes' Theorem** to dynamically update driver performance expectations:

$$\text{Posterior } P(A|B) \propto \text{Likelihood } P(B|A) \times \text{Prior } P(A)$$

```
+---------------------------------------+
|        FastF1 API / Live Data         |
+-------------------+-------------------+
                    |
                    v
+---------------------------------------+             +-----------------------------+
| FP3 / SQ Lap Times & Speed Traps      | ----------> | Bayesian ELO Updates (Form) |
+---------------------------------------+             +--------------+--------------+
                                                                     |
                                                                     v (Posterior ELO)
+---------------------------------------+             +--------------+--------------+
| Track Temperature & Rain Intensity    | ----------> | LightGBM Ranker (Qualifying)|
+---------------------------------------+             +--------------+--------------+
                                                                     |
                                                                     v (ML Predicted Grid)
+---------------------------------------+             +--------------+--------------+
| Starting Grid Order & Live Standings  | ----------> | Monte Carlo Race Simulator  |
+---------------------------------------+             +--------------+--------------+
                                                                     |
                                                                     v
                                                      +--------------+--------------+
                                                      | Win, Podium & DNF Prob %    |
                                                      +-----------------------------+
```

### A. Layer 0 — Season ELO Carryover (Cumulative Prior)
* **Data Source**: Iterates chronologically through all completed 2026 GPs. For each GP, updates driver ELO using qualifying head-to-head results (60% weight) and race finish head-to-head results (40% weight).
* **Formula**: `Posterior_ELO = Prior_ELO + K × weight × (Actual - Expected)` where K=24.
* **Output**: Cumulative driver ELO that reflects real 2026 season performance, not static pre-season estimates.

### B. Layer 1 — Dynamic Constructor Pace (Data-Driven)
* **Data Source**: Rolling 3-GP average of each team's best qualifying gap-to-pole position.
* **Output**: Dynamic `pace_offset` per constructor that replaces static hardcoded values.

### C. Prior — $P(A)$ (Season-Accumulated ELO + Driver History)
* **Data Source**: The cumulative Season ELO (Layer 0) combined with driver-level qualifying history (previous GP rank + rolling average of last 3 GPs) forms the structural prior fed into LightGBM LTR.

### D. Likelihood — $P(B|A)$ (Practice Performance Signal)
* **Data Source**: The normalized lap time delta of a driver in FP3/Sprint Qualifying relative to the fastest practice time of the weekend.

### E. Posterior — $P(A|B)$ (Combined Hybrid Score)
* **Data Source**: The final Hybrid Qualifying Score: `0.85 × ML_Ranker_Score + 0.15 × Practice_Pace_Score`. This determines Saturday's starting grid order, which feeds the Monte Carlo Race Simulator.

## 🚀 Installation & Quick Start

### 1. Clone the Repository
```bash
git clone https://github.com/fbudimannn/F1_PREDICTION.git
cd F1_PREDICTION
```

### 2. Create a Virtual Environment & Install Dependencies
```bash
python -m venv venv
source venv/bin/activate  # Linux/macOS
# or
venv\Scripts\activate     # Windows

pip install -r requirements.txt
```

### 3. Run the Streamlit Dashboard
```bash
streamlit run app.py
```

### 4. Run the Prediction Pipeline via CLI
You can execute the full qualifying and race prediction pipeline for any circuit directly from the terminal:
```bash
python run_pipeline.py --circuit canada --temp 28.0 --sims 10000
```

## 🧠 ML Training Pipeline

### Train/Test Split Strategy
* **Time-Series Cross-Validation**: Train on earlier seasons, validate on subsequent season
* **Fold 1**: Train 2022 → Validate 2023
* **Fold 2**: Train 2022-2023 → Validate 2024
* **Fold 3**: Train 2022-2024 → Validate 2025 (final test)

### Hyperparameter Tuning (Optuna)
```bash
python train_model.py
```
Search space: `n_estimators`, `learning_rate`, `num_leaves`, `max_depth`, `min_child_samples`, `reg_alpha`, `reg_lambda` over 30 Optuna trials.

### Anti-Overfitting Guards
* **Early stopping** and L1/L2 regularization
* **Overfit gap check**: |train_NDCG - test_NDCG| < 15%
* **Time-Series CV** prevents data leakage from future seasons

### Hybrid Architecture (2026 Regulation Compatibility)
Since 2026 introduces entirely new car regulations, the model uses a **hybrid approach**:
* **Structural Patterns** (from 2022-2025): How FP3 gaps correlate with qualifying gaps, how ELO predicts grid position
* **Current Season Data** (from 2026 only): Season ELO Carryover and Dynamic Constructor Pace use only real 2026 data

### Evaluation Metrics
| Metric | Description |
|---|---|
| NDCG@3 / @5 / @10 | Normalized Discounted Cumulative Gain for ranking |
| MAE | Mean Absolute Error for lap time prediction (seconds) |
| Overfit Gap | Difference between train and test NDCG@5 |
